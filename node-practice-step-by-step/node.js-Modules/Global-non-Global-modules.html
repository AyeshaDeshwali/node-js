Global modules (core modules) aur non-global modules (custom or third-party modules) ke beech me kuch antar hote hain: Global modules (core modules) Node.js ke sath aate hain aur bina require kiye use kiye ja sakte hain. Non-global modules ko explicitly require statement se import karna padta hai aur specific application requirements ke liye use kiye jate hain. Global modules built-in functionality provide karte hain, jabki non-global modules application customization aur third-party libraries ko include karne me help karte hain. jo modules aapne mention kiye hain, yani, os, assert, dns, punycode, readline, repl, dgram, url, querystring, string_decoder, timers, events, stream, zlib, util, crypto, cluster, process, console, domain, inspector, async_hooks, perf_hooks, v8, aur webcrypto, yeh sab core modules hote hain aur global modules (core modules) ki category me aate hain. Yani, in modules ko aap apne Node.js applications me bina kisi additional installation ya require statement ke use kar sakte hain. In modules ka istemal Node.js ke sath aata hai aur inka use cases specific tasks aur operations ke liye hota hai. Aap in modules ka istemal apne code me karke various functionality ko add kar sakte hain, jaise ki file system operations, network communication, assertion testing, event handling, cryptographic operations, and more.
